### 2.1 Huffman Compression
1. 首先需要定义压缩后文件的结构
+ 文件后缀: ***.huf***
> 文件结构：文件头+压缩后的比特流
+ 文件结构：
```
  --------------------------
  |         Header         |
  --------------------------
  |       Dictionary       |
  --------------------------
  |    Compressed bits     |
  --------------------------
```
+ 文件头：需要记录最后补齐多加的bit0个数
```
  --------------------------
  |  Dictionary Size(8B)   |   
  --------------------------
  |  additional bits(1B)   |
  --------------------------
  |    bits length(8B)     |
  --------------------------
```
2. 巨大缺陷：
+ 有一种明显的情况，即当文本长度十分短，但是内容十分“丰富”——即有很多不同的字符时，压缩效果会非常差。
> 此时编码表的长度可能比压缩文本本身还长
